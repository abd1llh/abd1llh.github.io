- Record current issues without changing behavior first.
- Capture mobile issues with screenshots.
- Document terminal problems, animation issues, performance issues, responsive issues, missing content, and future ideas.

## Phase 2 — Project Organization
- Keep planning files in `docs/`.
- Standardize file names.
- Remove unused files.

- Fix the mobile terminal.
- Fix keyboard overlap issues.
- Fix overflow.

- Fix responsive widths.
- Fix scrolling behavior.
